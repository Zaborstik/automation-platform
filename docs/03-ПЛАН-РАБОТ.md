# Общий план работ

## Принцип разделения

Работа разбита на **4 бота**. Каждый бот работает **исключительно** в своём модуле / модулях. Файлы между ботами **не пересекаются** -- один файл может трогать только один бот.

## Граф зависимостей между ботами

```
Бот 1 (core)          Бот 3 (agent/executor)     Бот 4 (knowledge)
    │                        │                          │
    │ (автономный)           │ (автономный)             │ (автономный)
    │                        │                          │
    ▼                        ▼                          ▼
  задачи 1.1-1.7          задачи 3.1-3.9            задачи 4.1-4.10
    │
    │  после завершения 1.1-1.3
    ▼
Бот 2 (api)
    │
    │ задачи 2.1-2.6, 2.9 -- ПАРАЛЛЕЛЬНО с Ботом 1
    │ задачи 2.7, 2.8, 2.10 -- ПОСЛЕ Бота 1
    ▼
  задачи 2.1-2.10
```

**Что можно делать параллельно:**
- Бот 1, Бот 3, Бот 4 -- полностью параллельно с первой минуты
- Бот 2 задачи 2.1-2.6, 2.9 -- параллельно с Ботом 1
- Бот 2 задачи 2.7, 2.8, 2.10 -- только после завершения Бота 1 (задачи 1.1-1.3)

## Правила для ботов

### Границы файлов (кто что трогает)

| Бот | Можно трогать | Нельзя трогать |
|-----|--------------|----------------|
| Бот 1 | `platform-core/src/**` | Всё остальное |
| Бот 2 | `platform-api/src/**` | Всё остальное |
| Бот 3 | `platform-agent/src/**`, `platform-executor/src/**` | Всё остальное (включая `playwright-server.js`!) |
| Бот 4 | `platform-knowledge/src/**`, `platform-knowledge/pom.xml` | Всё остальное |

### Правила написания кода
1. Каждая задача должна быть покрыта автотестами
2. Задачи должны выполняться в указанном порядке внутри бота
3. Не менять сигнатуры существующих public методов без необходимости
4. Новые методы добавлять как дополнительные (не ломая обратную совместимость)
5. Код на Java 21, стиль -- как в существующем проекте
6. Тесты: JUnit 5 + Mockito для unit, Spring Boot Test для интеграционных
