# mcp-servers/

> **Документ:** каталог Node.js **MCP-серверов** рядом с продуктом.  
> **К чему относится:** только каталог `mcp-servers/`; зеркало: [`docs/services/mcp-servers/README.ru.md`](../../docs/services/mcp-servers/README.ru.md).

Каталог для **MCP-серверов** (Model Context Protocol), запускаемых LLM-клиентами
(Claude Desktop, Cursor, Codex и т.п.) как локальные процессы. Каждый подкаталог —
самостоятельный артефакт со своим `package.json`, своим README и своими переменными
окружения.

## Почему это вне Maven-реактора

MCP-серверы здесь — это **Node.js-проекты**. Они:

- не собираются Maven и не включены в корневой [`pom.xml`](../pom.xml);
- имеют собственные зависимости через `npm` (`node_modules/` в этом каталоге игнорируется
  в [`.gitignore`](../.gitignore));
- запускаются LLM-клиентом по stdio, а не внутри Spring Boot;
- могут ходить как во внешние системы, так и в наш собственный `platform-api` — это зависит
  от конкретного сервера.

Это сознательное разделение: MCP — поверхность для LLM, она живёт рядом с продуктом,
но не внутри его сборки.

## Содержимое

| Папка                            | Что это                                                                                     | К какой системе ходит |
| -------------------------------- | ------------------------------------------------------------------------------------------- | --------------------- |
| [`dev-entity/`](./dev-entity/)   | Референсный MCP-сервер с CRUD-инструментами над внешним Entity REST API (стиль Jmix/CUBA). | Внешняя система (`MCP_ENTITY_BASE_URL`, по умолчанию `http://localhost:8080`). К нашему `platform-api` **не ходит**. |

## Установка зависимостей

Для каждого сервера — отдельно:

```bash
cd mcp-servers/<имя>
npm install
```

## Подключение к LLM-клиенту

См. README конкретного сервера. Общая схема — в `mcp.json` клиента прописать
`command: node`, `args: ["<полный путь>/server.js"]` и нужные `env`.

## Развитие

Если потребуется MCP-обёртка над нашим собственным `platform-api`
(`/api/entity-types`, `/api/plans`, `/api/actions`, `/api/workflows`), это будет
**отдельный** артефакт — параллельная папка `mcp-servers/platform-api/` (Node) или
новый Maven-модуль `platform-mcp` со `spring-ai-mcp-server-spring-boot-starter`.
Контракт у него будет свой; от `dev-entity` переиспользуется только подход
(JSON Schema для описания инструментов, retry на 401, набор операций), а не код.
